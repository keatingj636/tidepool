## 1. 项目概述

### 1.1 项目背景

ADHD（注意缺陷多动障碍）人群在日常任务管理中面临独特挑战：
- **时间盲 (Time Blindness)**：难以感知时间流逝，容易拖延
- **选择困难 (Choice Paralysis)**：任务过多时反而无法开始
- **执行功能障碍**：知道该做什么，但难以启动和维持行动
- **即时满足需求**：需要频繁的正向反馈来维持动力

### 1.2 项目愿景

为 ADHD 人群打造一款**极简、友好、高效率**的任务管理工具，帮助用户：
- 减少决策负担，专注于当下最重要的任务
- 建立时间感知，克服时间盲
- 获得持续的正向反馈，维持行动动力
- 灵活规划任务，无需严格的时间约束

### 1.3 项目目标

| 目标 | 指标 |
|------|------|
| 用户可快速添加任务 | 3 秒内完成添加 |
| 减少认知负担 | 界面元素精简，核心功能一目了然 |
| 提升任务完成率 | 通过待办机制和正向反馈 |
| 多设备同步 | 支持网页端随时访问 |

## 2. 用户画像

### 2.1 核心用户

| 属性 | 描述 |
|------|------|
| **主要人群** | 确诊 ADHD 患者 |
| **次要人群** | 有 ADHD 倾向（如拖延症、注意力不集中） |
| **年龄** | 18-45 岁 |
| **技术能力** | 中等，会使用常见 App |

### 2.2 用户痛点

1. **任务堆积造成焦虑** → 解决方案：待办只显示当天任务
2. **忘记任务截止时间** → 解决方案：智能提醒 + 日历可视化
3. **完不成任务产生挫败感** → 解决方案：无惩罚机制，只引导
4. **不知道从哪个任务开始** → 解决方案：专注模式（最多3件）

### 2.3 用户旅程

```
[产生想法] → [1秒快速添加] → [进入待办] → [完成获得正反馈] → [ streak 累计]
     ↑                                                                    │
     └────────────────────────────────────────────────────────────────────┘
```

## 4. 界面设计

### 4.1 设计原则

1. **极简主义** - 减少视觉元素，每次只关注最重要的事
2. **高对比度** - 深色背景 + 明亮强调色，易于识别
3. **大按钮** - 降低点击难度，减少误触
4. **即时反馈** - 每个操作都有明确的视觉/动画反馈
5. **无惩罚机制** - 不批评、不扣分，只有温和引导

### 4.2 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 背景 | 深蓝黑 | `#1a1a2e` |

| 主色调 | 珊瑚红 | `#e94560` |
| 强调色 | 琥珀橙 | `#f59e0b` |
| 成功色 | 翠绿 | `#4ade80` |
| 文字 | 浅灰白 | `#eaeaea` |

## 6. 数据模型

### 6.1 Task (任务)

```typescript
interface Task {
  id: string;           // 唯一标识 (UUID)
  title: string;       // 任务名称
  description?: string;// 任务描述 (可选)
  date: string;        // 日期 (YYYY-MM-DD)
  time?: string;       // 时间 (HH:mm)
  status: 'todo' | 'done';  // 状态
  createdAt: string;   // 创建时间 (ISO 8601)
  completedAt?: string;// 完成时间 (ISO 8601)
}
```

### 6.2 Settings (设置)

```typescript
interface Settings {
  streak: number;           // 当前连续完成天数
  mode: 'focus' | 'all';    // 当前显示模式
  theme: 'dark' | 'light'; // 主题 (未来)
}
```

### 6.3 数据流转

```
添加任务 → 存入 localStorage
    │
    ├── 未指定日期 → 默认今天 → 进入"待办"
    │
    ├── 指定日期 = 今天 → 进入"待办"
    │
    └── 指定日期 > 今天 → 进入"未来任务"

任务完成 → status = 'done' → streak + 1

次日检查 → 未完成任务 → date 改为今天 → 回到"待办"
```
